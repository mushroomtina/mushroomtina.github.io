  3. 性能相关

    使用vector时，如果知道vector的size，可以先reserve预分配内存，来避免随着vector不断添加元素而reallocate内存的现象。从而影响性能。

  4. debug coredump记录

    gdb调试core dump步骤：

    1. 编译debug包，并安装。
    2. 通过gdb执行用例。 注意：在gdb运行过程中，有可能会触发SIG32状态导致进程结束（报错为“Program received signal SIG32, Real-time event 32)。解决方案是在gdb窗口里敲入命令handle SIG32 nostop，来设置信号响应方式为nostop。

    core dump可能的产生原因：

    1. 数组index访问越界。
    2. 裸指针被其他shared_ptr持有，导致double free

  5. struct元素值更新失败？

    struct 是不是不允许update？初始化的时候就分配内存，不可更改

  6. gRPC错误码14（failed to connect to all addresses)问题排查:

    1. 检查grpc通信的目标地址及端口与实际是否相符。如client侧发起通信时所指定的ip和端口号是否与server实际ip及端口号一致。
    2. 检查目标机器服务是否正常工作。如server是否正常开启。
    3. 检查是否有设置代理。执行unset http_proxy; unset https_proxy。
